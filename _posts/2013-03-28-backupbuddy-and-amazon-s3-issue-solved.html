If you're using BackupBuddy to backup your wordpress installs, then you know how great it is.  If you've ever used it to move one of your wordpress sites, you really know what a life saver it can be.  I've been using it for a while now and have a good number of sites backed up on both dropbox and Amazon S3.  Until today, I never had an issue adding a new site to my collection.  When I was setting up this site (yes, my oldest site was not being backed up, hence all the missing images I need to fix) , I added BackupBuddy as I always do, and went to my Amazon Console and set up a new bucket.  I then proceeded to do a full back up and move it to to my offsite locations.  But this time, I noticed it didn't show up in my new S3 bucket.

After some fun Googling, I found an answer in BackupBuddy's support website.  Increase your php execution time, it must be timing out.  Hmm, ok, never had to do that before.  So I did, no luck.  Increased again, no luck.  Back to Google.  After about an hour of reading, I was beginning to think this was just one of those rare things that is only happening to me and nobody else in the world.  Then I found a hint in an AWS forum.

BackupBuddy only works with S3 if the bucket in created with the "US Standard" region selected.  My S3 was suddenly defaulting to Northern California for some reason. 

So hopefully this will help somebody Googling late at night because some default setting has changed that doesn't seem to be documented anywhere.  At least anywhere I have read in the last hour.
