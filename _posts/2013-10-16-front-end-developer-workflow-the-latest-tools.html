Workflows for front end devs are ever changing.  A few years ago, I didn't even think about work flows.  I used Coda, or some other text editor and FTP.  That was it.  Now, I have so many tools that help me with my daily tasks I don't do anything the same way anymore.  

My Current Tools

 Yeoman actually consists of 3 different tools that work together to help scaffold your web apps.  No more downloading this zip file or cloning git repos.  With Yeoman, you can spend time writing your app and not building the same framework again and again.

 Yo scaffolds your web app, building the directory structure, adding boilerplate files etc to quickly get straight to developing.  It also allows you to quickly build components for your app as well. You can download a number of different generators.  Whatever you're looking for, there is probably a generator for it.

 Grunt is used to test and build distributions of your web app.  It creates a server on you local dev machine that auto reloads your site as you edit files.

 Bower is a package manager, which means no more searching for and downloading zip files of this or that.

  Sublime Text - I've been using this text editor for a couple of years and don't think I will be changing any time soon.  It has a package manager that you can add which then makes adding plugins dead simple.  Some of those plugins are listed below.

SublimeGit  - This plugin gives you all the git tools available right in sublime, makes keeping your repo up to date and super clean. I switched from Tower to the command line to this plugin and it is by far the simplest git tool I've used yet.

Emmet  - This plugin allows you to write a ton of code super quickly by using short codes that translate to html and css.  Check out their page for a video. 

iTerm 2 - is a Terminal replacement which focuses on performance.  It includes features such as tabs, split screens and 256 colors.  It's a good alternative.

Beantalk  - is a private git repo that I use to store and deploy all of our websites for work and personal projects.  I have over 35 repos on beanstalk.  They are protected and are set up to deploy to server automatically on push. It's a nice alternative to GitHub when it comes to private repos.

 Github - is the answer for public repos.  github's trending pages are my new favorite pages to check daily. 

CodePen  - is a great place to try out the code  that has been floating around in your head.  It's great for sharing ideas and just showing off your latest css discovery.  They now have pro accounts with some great features.

DropBox  - Everyone knows drop box, but I have begun keeping my projects in drop box which allows me to leave work, come home and open my laptop and keep working without worrying about going to a repo.

Tools in the Trash

There are a few tools I thought were the thing a few months ago are now collecting virtual moth balls on my drive and I just don't need them anymore.  Tower for git, MAMP for hosting and Coda are all tools that are just too cumbersome to use in this new workflow.
